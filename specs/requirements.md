# 要件定義書: dialy (日記アプリケーション)

## 1. 目的
ユーザーが日々の出来事や感情を記録し、後から振り返ることができるシンプルな日記アプリケーションを提供する。

## 2. ターゲットユーザー
- 日々の出来事を手軽に記録したい人。
- 過去の自分の思考や感情を振り返りたい人。

## 3. 機能要件

### 3.1 ユーザー管理
- **認証**: ログインおよびログアウト機能。
- **マイページ**: プロフィール情報の閲覧および編集。
- **アカウント作成**: システム管理者が直接作成を行うため、ユーザーによる新規登録機能は提供しない。

### 3.2 日記管理
- **日記の作成**: タイトル、本文、日付（デフォルトは当日）、気分（Mood）を選択して保存。
- **日記の閲覧**:
    - 一覧表示（カレンダー形式またはリスト形式）。
    - 個別詳細表示。
- **日記の編集・削除**: 自分が作成した日記のみ編集・削除可能。
- **検索とフィルタリング**:
    - キーワード検索（タイトル、本文）。
    - 期間指定検索。
    - 気分別のフィルタリング。

### 3.3 カテゴリ/タグ管理 (拡張機能)
- 日記にタグを付与し、タグベースで日記を整理できる機能。

## 4. 非機能要件
- **整合性**: 日記データはユーザーに紐付けられ、他人の日記を閲覧・操作することはできないこと。
- **可用性**: シンプルな構成とし、起動速度とレスポンス性能を重視する。
- **保守性**: Spring Bootの標準的なレイヤードアーキテクチャを採用し、テストコードによって品質を担保する。

## 5. 技術スタック
- **言語**: Java 21
- **フレームワーク**: Spring Boot 3.x (最新版)
- **ビルドツール**: Maven
- **データベース**: H2 Database
- **ビュー**: Thymeleaf
- **テスト**: JUnit 5, Mockito, AssertJ
